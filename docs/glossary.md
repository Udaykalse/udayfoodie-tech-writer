# Glossary

## Overview

This glossary explains technical terms used throughout the **UdayFoodie Landing Page** project documentation.

---

## Asset

A static resource used by the website, such as an image, icon, or font.

**Example:** `assets/Images/img-1.webp`

---

## HTML (HyperText Markup Language)

The standard markup language used to structure web pages.

**Purpose:** Defines headings, paragraphs, images, buttons, and other page elements.

---

## CSS (Cascading Style Sheets)

A stylesheet language used to control the visual appearance of HTML elements.

**Purpose:** Defines colors, layouts, fonts, spacing, and responsive behavior.

---

## Responsive Design

A design approach that allows a website to adapt to different screen sizes and devices.

**Examples:**

- Desktop
- Laptop
- Tablet
- Mobile

---

## Semantic HTML

Using meaningful HTML elements that clearly describe their purpose.

**Examples:**

- `<header>`
- `<nav>`
- `<main>`
- `<section>`
- `<footer>`

Benefits include improved accessibility, readability, and SEO.

---

## Repository

A GitHub project that stores source code, documentation, and project history.

---

## Clone

Creating a local copy of a remote Git repository.

Example:

```bash
git clone https://github.com/Udaykalse/udayfoodie-landing-page.git
```

---

## Commit

A saved snapshot of project changes in Git.

Example:

```bash
git commit -m "Add project documentation"
```

---

## Push

Uploading local commits to a remote GitHub repository.

Example:

```bash
git push origin main
```

---

## Pull Request (PR)

A request to merge changes from one branch into another.

Pull Requests are commonly used for code reviews and collaboration.

---

## Branch

An independent line of development in a Git repository.

Example:

- `main`
- `feature/documentation`

---

## Markdown

A lightweight markup language used for writing documentation.

Files use the `.md` extension.

Examples:

- `README.md`
- `installation.md`
- `features.md`

---

## Documentation

Written information that explains how a project works, how to install it, use it, and maintain it.

Good documentation helps developers understand a project quickly.

---

## Static Website

A website built using HTML, CSS, and optional JavaScript, without a backend server or database.

---

## Summary

This glossary provides definitions for common terms used throughout the project documentation, helping new contributors and readers understand the technical concepts more easily.
