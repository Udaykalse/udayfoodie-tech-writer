# Assets Documentation

## Overview

This document explains how static assets are organized and managed in the **UdayFoodie Landing Page** project.

Static assets include images and other resources required for the website to function correctly.

---

# Assets Directory

```text
assets/
└── Images/
    ├── delivery__img.webp
    ├── Fresh-ingredients2.webp
    ├── ian-dooley-d1UPkiFd04A-unsplash.jpg
    ├── img-1.webp
    ├── Simple-cooking.webp
    └── Variety-of-meals.webp
```

---

# Purpose of the Assets Folder

The `assets/` directory keeps all static resources separate from source code.

Benefits include:

- Better project organization
- Easier maintenance
- Improved readability
- Simplified asset management

---

# Image Assets

The project uses images for:

- Hero banners
- Food items
- Delivery illustrations
- Informational sections
- Visual enhancement

---

# Supported Image Formats

The project currently includes:

| Format | Purpose |
|---------|----------|
| `.webp` | Optimized web images |
| `.jpg` | High-quality photographs |

Additional supported formats:

- PNG
- SVG
- JPEG

---

# Naming Convention

Image filenames should be:

- Descriptive
- Lowercase where possible
- Easy to understand
- Free of unnecessary spaces

Example:

```text
hero-banner.webp
menu-item-1.webp
customer-review.jpg
```

---

# Best Practices

When adding new assets:

- Optimize image size before uploading.
- Use descriptive filenames.
- Avoid duplicate images.
- Keep related assets together.
- Remove unused files.

---

# Performance Considerations

Optimized assets improve:

- Page load speed
- Browser performance
- User experience
- SEO

Recommendations:

- Prefer WebP for web images.
- Compress large files.
- Resize images before adding them.
